Описание формата данных
Минутные данные из Binance API (endpoint /api/v3/klines) представляют собой список, где каждая свеча содержит 12 элементов. Для удобства в CSV сохраняются только основные поля: timestamp, open, high, low, close, volume. Полный формат свечи:

timestamp (индекс 0): Время открытия свечи (Unix timestamp в миллисекундах, например, 1577836800000). В CSV преобразуется в читаемый формат (например, 2020-01-01 00:00:00).
open (индекс 1): Цена открытия (например, 129.14).
high (индекс 2): Максимальная цена за минуту (например, 129.25).
low (индекс 3): Минимальная цена за минуту (например, 128.90).
close (индекс 4): Цена закрытия (например, 129.10).
volume (индекс 5): Объем торгов в ETH (например, 123.45).
close_time (индекс 6): Время закрытия свечи (Unix timestamp в миллисекундах, например, 1577836859999). Не сохраняется в CSV.
quote_volume (индекс 7): Объем торгов в USDT (например, 15987.65). Не сохраняется в CSV.
trades (индекс 8): Количество сделок (например, 150). Не сохраняется в CSV.
taker_buy_volume (индекс 9): Объем покупок маркет-тейкерами в ETH (например, 80.12). Не сохраняется в CSV.
taker_buy_quote_volume (индекс 10): Объем покупок маркет-тейкерами в USDT (например, 10345.78). Не сохраняется в CSV.
ignore (индекс 11): Зарезервировано (игнорируется, обычно 0). Не сохраняется в CSV.
